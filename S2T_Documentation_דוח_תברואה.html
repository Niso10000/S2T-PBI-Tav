1 Overview

This is a municipal sanitation operations monitoring report in Hebrew. It tracks, measures, and scores multiple sanitation service domains across geographic regions (נפות = districts). Each domain has its own data source, with scores normalized for cross-domain comparison.

ℹ️
Confidence levels: High for structure & report layer · Medium for measure logic (inferred from names) · Low for DAX expressions & relationship cardinality (DataModel binary inaccessible)

Key Subject Areas

  • Daily operational scoring per region and per vehicle/cart/truck
  • Performance trends over time (% score change vs. prior period)
  • Execution vs. planning compliance (bins collected vs. planned)
  • Garbage tonnage monitoring at disposal sites
  • Public complaint SLA tracking (open / closed / overdue / reopened)
  • Events and sanitation incident logging

6.3 OData / Dynamics CRM (incidents)

Connects to Jerusalem CRM (.api.crm4.dynamics.com) via OData v9.1. Applies date type conversion and filters by division GUID:

let Source = OData.Feed( "https://.api.crm4.dynamics.com/api/data/v9.1/incidents" & "?$select=incidentid,pws_status,pws_slastatus,createdon,_pws_departmentid_value,..." & "&$filter=(pws_casetype eq 2 or pws_casetype eq 3)" & " and _pws_divisionid_value eq ''" & " and createdon ge 2022-01-01", null, [Implementation="2.0"] ), #"Changed Type" = Table.TransformColumnTypes(Source, {{"createdon", type datetime}}), #"Extracted Date" = Table.TransformColumns(#"Changed Type", {{"createdon", DateTime.Date, type date}}) in #"Extracted Date"

6.4 OData — pws_casestatus (Option Set expansion)

Reads a GlobalOptionSet definition and expands nested records to extract Hebrew status labels, filtering out Arabic entries:

let Source = OData.Feed("https://.crm4.dynamics.com/.../GlobalOptionSetDefinitions(feb2cbe5-...)"), Options = Source[Options], #"Expanded Label" = Table.ExpandRecordColumn(Options, "Label", {...}), #"Renamed Columns"= Table.RenameColumns(..., {{"Label.LocalizedLabels.Label", "סטטוס"}}), #"Filtered Rows" = Table.SelectRows(#"Renamed Columns", each ([סטטוס] <> "بالمعالجة" and [סטטוס] <> "نهاية المعالجة")) ← removes Arabic entries in #"Filtered Rows"

11 Improvement Suggestions

🏗 Model Structure
  • Disable / remove LocalDateTable_c2183... — all date hierarchies should route through DimDate
  • Rename ‏‏EventsData → EventsData — remove the leading invisible Unicode characters
  • Investigate TAV_Regions_CRM — if unused, remove from model to reduce size
  • Consider consolidating V_TAV_Regions / TAV_Regions_CRM / TAV_Regions_Gr into a unified region dimension
📐 DAX Optimization
  • Implement Calculation Groups — the ציון ממוצע / כל הנפות / % שינוי triplet is duplicated 4× across domains. A Calculation Group eliminates this and makes maintenance trivial
  • Single טווח תאריכים measure — currently duplicated in multiple fact tables as _It, _Gb, _Gr variants. One measure in DimDate using CALCULATE is sufficient
  • Formalize QtyPolygonExit — currently used as an implicit aggregation Sum(V_TAV_FactDailyRoute.QtyPolygonExit) in visuals. Should be a named explicit measure
🏷 Naming Standardization
  • Standardize ציון ממוצע_GB → ציון ממוצע_Gb (capitalize consistently throughout V_TAV_GB_FactDailyRoute)
  • Fix typo: ממןצע → ממוצע in affected measure name
  • Standardize שעות_עבודה (underscore) → שעות עבודה_Ge (space + suffix) — or remove the duplicate
📄 Documentation Gaps
  • No measure descriptions in the model — critical for developers. Add descriptions to all measures in Tabular Editor or Power BI Desktop
  • RLS setup documentation missing — TAV_Users population process and IsVisable flag management should be documented separately
  • Scoring algorithm not documented — TAV_Marks defines the rules, but the calculation formula linking marks → weighted average score is not visible
  • Source system refresh schedule unknown — no documentation on data freshness SLAs
